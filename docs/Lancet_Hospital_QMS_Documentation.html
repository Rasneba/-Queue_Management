The Lancet General Hospital Queue Management System is a full-stack web application designed to digitize and streamline the patient flow from check-in through doctor consultation. The system replaces manual paper-based queue management with an intelligent, real-time dashboard that supports Amharic, Afaan Oromoo, and English — including text-to-speech voice announcements using female Ethiopian voices.

Key capabilities include self-service kiosk check-in, automatic patient calling, rule-based clinical triage, real-time waiting board displays, doctor shift management, and a comprehensive back-office admin panel for staff registration and performance reporting.

📝 Stage Details

1. Self Check-in

Patient enters name, age, gender, symptoms. Ticket P-N auto-generated. No triage at this stage.

2. Waiting Board

Patients displayed by ticket order. When audio enabled, next patient is auto-called with Amharic TTS.

3. Reception

Auto-calls next patient. Receptionist can Full Triage, Skip Triage, Recall, Hold, or Assign Doctor directly.

4. Triage

Rule-based keyword triage assigns Emergency/High/Medium/Low priority + department routing.

5. Doctor

Doctor starts shift, calls patient to room, treats, then completes & discharges.

🔄 Patient Status Flow

// Patient lifecycle states Waiting → Called → Serving → Completed | | +--> NoShow +--> Waiting (Hold back) // Status transitions: Check-in → Waiting Call → Called (assignedRoom set) Serve → Serving Complete → Completed (+ completedTime) Hold → Waiting (back to queue) No-Show → NoShow

Triage Priority Levels

Emergency (5) High (4) Medium (3) Low (1-2)

4. Features by Role

Each hospital role has a dedicated console with tailored functionality

📋

Reception Console

Counter-based workflow
  • Auto Auto-calls next waiting patient by ticket order
  • Triage Full Triage with symptom editing + AI scoring
  • Triage Skip Triage (use existing data)
  • Doctor Assign to active doctor only (from DB sessions)
  • Control Recall, Hold, No-Show patient actions
  • TTS Amharic voice announcements
  • Auth Password-protected counter login
🩺

Triage Team

Clinical priority assessment
  • Assess Rule-based keyword triage (no AI dependency)
  • Priority Emergency / High / Medium / Low scoring
  • Route Auto-assign department (Cardiology, Pediatrics, etc.)
  • Vitals Suggested vitals to measure per condition
  • Explain Clinical justification + precautions
  • Override Manual triage adjustment by doctor
  • Flow Forward to doctor after triage
⚕

Doctor Dashboard

Consultation management
  • Shift Start/End shift (DB-backed, not localStorage)
  • Room Assign consulting room per shift
  • Queue View patients by status tabs
  • Call Call patient to room with TTS announcement
  • Treat Start treatment → Complete & discharge
  • Override Manual triage priority adjustment
  • Stats Shift duration timer + patients treated count
🏢

Back-office Admin

/backoffice
  • Staff Register Reception / Triage / Doctor staff
  • Auth Password-protected login per staff member
  • Reports Triage performance (priority distribution)
  • Reports Reception throughput (check-ins by hour)
  • Reports Doctor performance (patients treated, shifts)
  • Filters Daily / Weekly / Monthly / Custom date range
  • Staff Filter reports by specific staff member

5. Text-to-Speech & Amharic Voice System

Real-time multilingual voice announcements using Microsoft Edge TTS

🔊 Voice Engine

  • Engine Microsoft Edge TTS (edge-tts Python package)
  • Voice am-ET-MekdesNeural (Female Amharic)
  • Voice om-ET-AmehaNeural (Female Afaan Oromoo)
  • Voice en-US-JennyNeural (Female English)

6. Back-office System

Separate admin panel at /backoffice with staff auth and reporting

🔒 Login

  • Staff name + password authentication
  • Session stored in localStorage
  • Auto-redirect if already logged in
  • Role-based access control

👥 Staff Management

  • Register with Name + Role + Password + Department
  • Three roles: Reception, Triage, Doctor
  • Filter by role, view registration date
  • Remove staff with confirmation

📊 Reports

  • Triage: priority distribution + completion rate
  • Reception: check-ins by hour + status breakdown
  • Doctor: patients treated + shift hours + avg/shift
  • Date filters + staff selector

Set environment variables echo "DATABASE_URL=postgresql://..." > .env.local # 4. Start everything (Next.js + TTS server) npm run dev # 5. 11. Performance & Security

Optimized for speed and reliability

⚡ Performance Optimizations

  • Smart Polling Skip re-renders when data unchanged (JSON diff)
  • Memoization useMemo for patient filters, counts, derived data
  • Code Splitting All views lazy-loaded via next/dynamic
  • Serverless DB Neon PostgreSQL scales to zero when idle
  • Edge Runtime API routes run at the edge for low latency
  • TTS Caching Audio cached on client to reduce API calls

🔒 Security

  • Auth Password-protected reception counters
  • Auth Password-protected back-office access
  • SSL Neon DB connection uses SSL mode
  • Env Database credentials in .env.local (not committed)
  • Input All API routes validate required fields
  • HTTPS Ready for HTTPS in production
